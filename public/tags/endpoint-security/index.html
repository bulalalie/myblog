Differences in drivers, kernel extension, and system extension Overview Create drivers, system extensions, and kernel extensions for specific low-level system services. A DriverKit extension (dext) manages the communication between your company’s hardware device and the rest of the system. A system extension implements features that require kernel-level cooperation, such as custom security and network behaviors. A kernel extension (kext) supports any low-level services that cannot be implemented using a dext or system extension.
